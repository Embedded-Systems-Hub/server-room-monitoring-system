## MISRA C:2012 Deviations

This file documents all MISRA C:2012 rule deviations. Each entry classifies
the deviation as either a **justified deviation** or a **false positive**.

---

### Rule 10.1, 10.4, 10.8, 12.1, 15.5 — bmp280.c: BMP280_CompensateTemp()
Category: Justified deviation
Deviation: Bitwise and shift operators applied to signed int32_t operands;
casting composite expressions; operator precedence; multiple return points.
Justification: The temperature compensation formula is taken directly from
the Bosch BMP280 datasheet (section 8.2). The signed intermediate values
are required for correct compensation. Converting to unsigned types would
alter the algorithm behavior and produce incorrect results.

---

### Rule 17.3 — bmp280.c, app_tasks.c
Category: False positive
Deviation: Implicit function declaration for HAL and CMSIS-RTOS2 functions.
Justification: The STM32 HAL and CMSIS-RTOS2 headers are only available
when building with the ARM toolchain and STM32 SDK. cppcheck running on
the host cannot resolve these includes. The functions are properly declared
in the SDK headers used during firmware compilation.

---

### Rule 21.6 — app_tasks.c
Category: Justified deviation
Deviation: Use of snprintf() from <stdio.h>.
Justification: snprintf is used for UART diagnostic logging only. This
system is not safety-critical. A MISRA-compliant custom formatter would
add complexity without benefit at this project scope.

---

### Rule 21.1 — Core/Inc/i2c.h, usart.h, main.h
Category: False positive
Deviation: Include guards use reserved identifier patterns (__NAME_H__).
Justification: These files are generated by STM32CubeMX and cannot be
modified without being overwritten on next code generation.

---

### Rule 8.4, 11.8 — main.c
Category: False positive / HAL API limitation
Deviation: Missing compatible declarations; const cast removal in HAL_UART_Transmit call.
Justification: HAL_TIM_PeriodElapsedCallback is a HAL weak symbol with a
fixed signature. HAL_UART_Transmit takes uint8_t* (non-const) by API design;
const correctness cannot be enforced at the call site without modifying the HAL.

---

### Rule 10.8 — app_tasks.c
Category: Justified deviation
Deviation: Cast of composite expression (long)(int32_t / 100) for snprintf.
Justification: int32_t maps to int on ARM Cortex-M4. The (long) cast is
required to match the %ld format specifier portably across host and target.
A MISRA-compliant approach would use a custom print function.
